# UI设计参考指南

## 概述
本文档提供了UI设计的参考资料、最佳实践和案例分析，用于指导UI Designer Agent的设计输出，确保输出的UI设计方案符合现代设计标准和行业最佳实践。

---

## 第一部分：设计系统基础

### 1. 颜色理论

#### 1.1 色彩搭配方案

**互补色搭配（Complementary Colors）**
- 定义：相对色轮上相对的颜色
- 特点：对比强烈，充满活力
- 用途：创建有视觉冲击的界面
- 示例：
  - 红 (#FF6B6B) + 青 (#4ECDC4)
  - 橙 (#FF9500) + 蓝 (#007AFF)
  - 紫 (#8B5FBF) + 黄 (#FFE66D)

**类似色搭配（Analogous Colors）**
- 定义：色轮上相邻的颜色
- 特点：和谐统一，舒适柔和
- 用途：创建一致性高的界面
- 示例：
  - 蓝 (#007AFF) + 紫 (#5856D6) + 粉红 (#FF2D55)
  - 绿 (#34C759) + 青 (#4ECDC4) + 蓝 (#007AFF)

**三角形搭配（Triadic Colors）**
- 定义：色轮上均匀分布的三种颜色
- 特点：平衡、生动、专业
- 用途：创建视觉效果均衡的界面
- 示例：
  - 红 (#FF3B30) + 黄 (#FFD700) + 蓝 (#007AFF)
  - 绿 (#34C759) + 紫 (#9500FF) + 橙 (#FF9500)

---

#### 1.2 颜色在不同场景的应用

**游戏界面颜色应用**

| 场景 | 推荐颜色 | 原因 | 示例 |
|------|--------|------|------|
| 主按钮/CTA | 高饱和度 | 吸引注意，鼓励点击 | #007AFF (iOS蓝) |
| 成功反馈 | 绿色系 | 与"通过"的心理关联 | #34C759 |
| 警告/重要 | 红色/橙色 | 提醒用户 | #FF3B30 或 #FF9500 |
| 禁用状态 | 灰色系 | 表示不可用 | #A9A9A9 |
| 中立/信息 | 蓝色系 | 专业、可信 | #007AFF |
| 背景 | 浅色/中性 | 降低疲劳 | #FFFFFF 或 #F7F9FC |

**娱乐/社交游戏配色**
- 主色：暖色（红、橙、黄）- 活跃、热情
- 辅色：冷色（蓝、青、紫）- 平衡、稳定
- 强调色：高饱和度 - 吸引注意

**企业级应用配色**
- 主色：深蓝、深灰 - 专业、信任
- 辅色：中等蓝、浅灰 - 平衡
- 强调色：单一对比色 - 引导

---

#### 1.3 颜色无障碍性

**色盲友好设计**

| 类型 | 占比 | 建议 |
|------|------|------|
| 红绿色盲 | 8% (男) | 避免纯红+纯绿组合 |
| 蓝黄色盲 | <1% | 少见，注意蓝黄对比 |
| 完全色盲 | <1% | 使用高对比度灰度 |

**对比度标准（WCAG 2.1）**
- **AAA级**（推荐）：7:1 - 最高可访问性
- **AA级**（最低要求）：4.5:1 (普通文本)、3:1 (大文本)
- **不符合**：<3:1 - 不可用

**检查工具**：
- WebAIM对比度检查器：https://webaim.org/resources/contrastchecker/
- Colblindor模拟器：https://www.colblindor.com/

### 2. 排版系统

#### 2.1 字体大小层级

**推荐的排版尺度（基于8px网格）**

| 用途 | 大小 | 行高 | 字重 | 示例 |
|------|------|------|------|------|
| 超大标题 | 32px | 40px | 700 | 游戏标题 |
| 大标题 | 24px | 32px | 700 | 页面标题 |
| 中标题 | 20px | 28px | 600 | 卡片标题 |
| 小标题 | 16px | 24px | 600 | 组件标题 |
| 正文 | 14px | 22px | 400 | 描述文本 |
| 辅助 | 12px | 18px | 400 | 标签、提示 |
| 极小 | 10px | 16px | 400 | 时间戳、备注 |

**最佳实践**
- 最小字体不低于12px（移动端）
- 标题字重600以上
- 正文字重400（Light）或500（Regular）
- 使用行高 = 字体大小 × 1.4-1.6

---

#### 2.2 推荐字体

**web字体（免费）**
- **系统字体栈**（最快）：
  ```css
  font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'Roboto', sans-serif;
  ```
- **Google Fonts推荐**：
  - 标题：Poppins, Montserrat, Inter
  - 正文：Open Sans, Roboto, Source Han Sans

**中文字体**
- **系统字体**：微软雅黑、苹方
- **Google字体**：Noto Sans CJK
- **阿里系字体**：阿里巴巴普通话

### 3. 间距与布局系统

#### 3.1 8px栅格系统

**间距规范（8px倍数）**

| 间距 | 像素 | 用途 |
|------|------|------|
| xs | 4px | 组件内部极小间距 |
| sm | 8px | 组件内部间距、紧凑 |
| md | 16px | 标准间距 |
| lg | 24px | 大组件间距 |
| xl | 32px | 部分间距 |
| xxl | 40px | 大间距、页面段落 |

**应用示例**
- 按钮内边距：8px 16px（上下8px，左右16px）
- 卡片内边距：16px（所有方向）
- 卡片间距：16px
- 页面边距：16-24px

---

#### 3.2 响应式断点

**常见断点定义**

| 设备 | 宽度 | 栅格列数 | 边距 |
|------|------|--------|------|
| 手机 | <480px | 4 | 16px |
| 小平板 | 480-768px | 8 | 16px |
| 平板 | 768-1024px | 12 | 24px |
| 桌面 | >1024px | 12 | 24-32px |

### 4. 组件设计规范

#### 4.1 按钮组件

**尺寸规范**

```
大按钮（Large）
- 高度：48px
- 内边距：12px 24px
- 字体：16px, 600
- 用途：主要操作

标准按钮（Default）
- 高度：40px
- 内边距：10px 16px
- 字体：14px, 600
- 用途：常规操作

小按钮（Small）
- 高度：32px
- 内边距：6px 12px
- 字体：12px, 600
- 用途：次要操作
```

**按钮状态**
- Normal（正常）：颜色正常
- Hover（悬停）：颜色加深或透明度减低
- Active（按下）：颜色更深或添加阴影
- Disabled（禁用）：灰色，不可点击

**按钮类型**
- **Primary**：主操作（核心色）
- **Secondary**：次操作（边框或浅色背景）
- **Danger**：危险操作（红色）
- **Ghost**：文本按钮（无背景）

---

#### 4.2 卡片组件

**卡片规范**

```
布局：
- 边距：16px
- 边框圆角：8px
- 投影：0 2px 8px rgba(0,0,0,0.1)

内容：
- 标题：16px, 600
- 描述：14px, 400
- 行间距：间距系统

状态：
- Normal：白色背景
- Hover：浅色背景或阴影增强
- Selected：边框或背景高亮
```

**卡片示例**
- 房间卡片：房间号、玩家数、状态
- 房产卡片：房产名称、价格、租金
- 玩家卡片：头像、名称、等级

---

#### 4.3 输入框组件

**输入框规范**

```
尺寸：
- 高度：40px
- 内边距：8px 12px
- 字体：14px

边框：
- 宽度：1px
- 颜色：#D0D0D0
- 圆角：4px

状态：
- Focus：边框色变为主色，阴影
- Error：边框色为红色，错误提示
- Disabled：背景灰色，不可编辑

占位符：
- 颜色：#999999
- 不显示真实内容
```

### 5. 动画与微交互

#### 5.1 推荐的动画参数

**持续时间**
- 快速交互：100-150ms（按钮点击）
- 中等转换：200-300ms（页面切换）
- 慢速动画：400-500ms（入场动画）

**缓动函数（Easing）**
```
ease-in-out：标准缓动（推荐）
cubic-bezier(0.4, 0, 0.2, 1)

ease-out：快速进入，缓速退出
cubic-bezier(0, 0, 0.2, 1)

ease-in：缓速进入，快速退出
cubic-bezier(0.4, 0, 1, 1)
```

**常见微交互**
- **按钮点击**：缩放 0.95-1.0
- **页面进入**：淡入或滑入
- **模态框**：背景淡入，内容放大
- **加载状态**：旋转或脉冲动画

---

#### 5.2 游戏特殊动画

**骰子动画**
- 投掷：旋转+ 放大 + 掉落效果
- 持续时间：600-800ms
- 最终显示：停止并放大突出结果

**棋子移动**
- 移动：平滑的位移动画
- 持续时间：500-800ms
- 缓动：ease-in-out

**卡片翻转**
- 3D翻转效果
- 持续时间：400ms
- 配合音效提升体验

## 第二部分：设计案例

### 1. 在线大富翁游戏UI设计案例

#### 1.1 颜色方案
```json
{
    "primary": "#FF6B6B",      // 活跃的红色
    "secondary": "#4ECDC4",    // 平衡的青绿色
    "accent": "#FFE66D",       // 突出的黄色
    "background": "#F7F7F7",   // 中性背景
    "text": "#2C3E50",         // 深灰文字
    "success": "#34C759",      // 绿色反馈
    "danger": "#FF3B30"        // 红色警告
}
```

**应用**：
- 主按钮：#FF6B6B（开始游戏、购买房产）
- 卡片背景：#4ECDC4（房间卡片、道具卡片）
- 成功提示：#34C759（交易完成、升级）
- 页面背景：#F7F7F7（主背景）

---

#### 1.2 页面结构
```
1. 首页
   - 游戏Logo + 品牌形象
   - 开始游戏/登录按钮
   - 新闻/活动公告

2. 游戏大厅
   - 房间列表（卡片形式）
   - 创建房间按钮
   - 好友/排行榜
   - 个人信息

3. 房间内
   - 玩家列表
   - 游戏规则设置
   - 开始/取消按钮
   - 聊天框

4. 游戏进行中
   - 棋盘展示（中心）
   - 玩家信息栏（周围）
   - 骰子与操作按钮
   - 聊天与表情

5. 游戏结算
   - 排名与奖励
   - 统计信息
   - 分享与回放
```

---

#### 1.3 核心组件
```
导航类：
- 底部导航栏（大厅、游戏、个人中心）
- 顶部标题栏（返回、设置）

输入类：
- 房间名称输入
- 聊天输入框
- 规则参数滑块

展示类：
- 房间卡片（房号、状态、玩家数）
- 玩家卡片（头像、名称、等级、金币）
- 房产卡片（名称、价格、租金、所有者）

操作类：
- 开始/结束按钮
- 骰子按钮（特殊交互）
- 购买/交易按钮
- 聊天/表情按钮

反馈类：
- Toast提示（操作完成）
- 模态框（确认、选择）
- 加载动画（等待）
```

### 2. 设计对比分析

#### 2.1 现代游戏UI vs 传统游戏UI

**现代游戏UI特点**
- 扁平设计，极简美学
- 明亮的配色，高对比度
- 无缝的响应式设计
- 流畅的微动画
- 清晰的视觉层级

**传统游戏UI特点**
- 拟物化设计，3D效果
- 深色系配色
- 固定分辨率
- 少量动画
- 装饰元素多

**选择建议**：
- 休闲社交游戏：选择现代UI
- 竞技/RPG游戏：可融合两种风格
- 移动优先：选择现代UI（响应式友好）

---

#### 2.2 配色风格对比

| 风格 | 主色 | 特点 | 适用 |
|------|------|------|------|
| 热情运动 | 红+橙+黄 | 活跃、年轻 | 社交游戏 |
| 专业冷静 | 深蓝+灰 | 稳定、信任 | 企业应用 |
| 精致高端 | 深灰+金 | 奢华、优雅 | 高端游戏 |
| 清新文艺 | 绿+粉+蓝 | 舒适、温暖 | 休闲游戏 |

## 第三部分：设计检查清单

### 上线前检查清单

#### 颜色检查
- [ ] 主色与辅色对比度 >= 4.5:1
- [ ] 文字与背景对比度 >= 4.5:1
- [ ] 色盲模拟测试通过
- [ ] 颜色搭配和谐
- [ ] 颜色数量不超过5种主色

#### 排版检查
- [ ] 最小字体 >= 12px
- [ ] 标题字重 >= 600
- [ ] 行高 >= 字体大小 × 1.4
- [ ] 标题、正文、辅助文字清晰区分
- [ ] 字体族使用系统字体或Google字体

#### 布局检查
- [ ] 间距基于8px栅格
- [ ] 响应式设计测试（手机/平板/桌面）
- [ ] 内容留白充足
- [ ] 对齐一致
- [ ] 组件尺寸规范化

#### 组件检查
- [ ] 按钮最小尺寸 40×40px（可交互区域）
- [ ] 输入框高度 >= 40px
- [ ] 卡片有清晰的边界
- [ ] 所有组件支持4个状态（normal, hover, active, disabled）
- [ ] 组件风格一致

#### 交互检查
- [ ] 所有可点击元素有hover状态
- [ ] 加载状态有反馈动画
- [ ] 错误有明确的错误提示
- [ ] 成功有反馈提示
- [ ] 过渡动画平滑

#### 无障碍检查
- [ ] 色盲友好
- [ ] 对比度充足
- [ ] 字体大小可读
- [ ] 按钮可通过键盘访问
- [ ] 有焦点状态指示

## 最佳实践总结

### ✅ DO（应该做）
- 使用系统字体或Google字体
- 基于8px栅格设计
- 遵循WCAG 2.1 AA标准
- 设计时考虑响应式
- 组件化和系统化设计
- 提供4个以上的组件状态
- 使用清晰的视觉层级
- 测试色盲友好性

### ❌ DON'T（不应该做）
- 使用奇特的、难以阅读的字体
- 任意设置间距和尺寸
- 忽视无障碍性设计
- 只为桌面设计
- 过度装饰和复杂
- 忽略组件状态设计
- 创建视觉混乱
- 使用纯红+纯绿的搭配
