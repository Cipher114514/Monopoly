# UI Designer Agent 提示词文档

## 概述
本文档记录了UI Designer Agent（UI设计Agent）的提示词设计和演进历史。UI Designer Agent负责基于PRD生成结构化的UI设计方案，包括颜色方案、页面结构、组件设计和交互设计。

---

## 核心提示词

### 主提示词模板

```python
ui_prompt = f"""你是资深UI/UX设计师。

## 产品信息
产品名称：{prd_data['product_name']}
功能列表：{', '.join(prd_data['features'])}
目标用户：{prd_data['target_users']}

## 任务
为该产品设计完整的UI界面，包含以下信息：
1. color_scheme: 颜色方案（包含主色、辅色、强调色）
2. pages: 页面列表（核心页面名称）
3. components: 组件列表（每个组件包含name, type, description）
4. interactions: 交互列表（核心交互行为）

## 颜色方案格式
{{
    "primary": "#HEX",
    "secondary": "#HEX",
    "accent": "#HEX",
    "background": "#HEX",
    "text": "#HEX"
}}

## 组件格式
[{{
    "name": "组件名称",
    "type": "button|input|card|etc",
    "description": "组件描述"
}}]

## 重要
请直接输出纯 JSON 格式，不要使用 markdown 代码块包裹，不要添加任何其他文字。

示例格式：
{{
    "color_scheme": {{"primary": "#007AFF", "secondary": "#5856D6", "accent": "#FF9500", "background": "#FFFFFF", "text": "#000000"}},
    "pages": ["首页", "游戏大厅", "游戏界面"],
    "components": [{{"name": "开始游戏按钮", "type": "button", "description": "触发游戏开始"}}],
    "interactions": ["点击按钮开始游戏", "拖拽棋子移动"]
}}
"""
```

## 提示词组件详解

### 1. 角色设定 (Role Setting)
```
你是资深UI/UX设计师。
```

**目的**：
- 为LLM设定专业的设计角色
- 确保输出具有设计专业性
- 引导LLM考虑用户体验

**优化建议**：
- 可强调特定设计风格：「Material Design风格的UI/UX设计师」、「iOS Design风格的UI/UX设计师」
- 可指定设计经验领域：「游戏UI设计师」、「企业级应用UI设计师」

---

### 2. 产品上下文 (Product Context)
```
## 产品信息
产品名称：{prd_data['product_name']}
功能列表：{', '.join(prd_data['features'])}
目标用户：{prd_data['target_users']}
```

**目的**：
- 提供UI设计的基础信息
- 确保设计符合产品特性
- 针对目标用户的设计考量

**数据来源**：
- 来自Requirements Agent的PRD输出
- `prd_data`字段包含结构化数据

**示例**：
```

功能列表：多人实时对战, 自定义游戏规则, 房产购买与交易系统, 语音聊天与表情互动
目标用户：18-45岁，喜欢休闲社交游戏的用户
```

### 3. 设计要素（Design Elements）

#### 3.1 颜色方案 (Color Scheme)
```
## 颜色方案格式
{
    "primary": "#HEX",
    "secondary": "#HEX",
    "accent": "#HEX",
    "background": "#HEX",
    "text": "#HEX"
}
```

**设计指导**：
- **primary（主色）**：品牌核心色，用于主要操作和强调
  - 示例：#007AFF (iOS蓝)、#FF6B6B (现代红)
  - 作用：按钮、标题、链接
  
- **secondary（辅色）**：补充色，用于次要操作
  - 示例：#5856D6 (紫色)、#4ECDC4 (青绿色)
  - 作用：卡片、分组、装饰
  
- **accent（强调色）**：突出色，用于吸引注意
  - 示例：#FF9500 (橙色)、#34C759 (绿色)
  - 作用：警告、成功、特殊状态
  
- **background（背景色）**：界面底色
  - 示例：#FFFFFF (白色)、#F7F9FC (浅灰)
  - 作用：主背景、卡片背景
  
- **text（文字色）**：主要文本颜色
  - 示例：#000000 (黑色)、#2C3E50 (深灰)
  - 作用：标题、正文、标签

**最佳实践**：
- 颜色应有充分的对比度（WCAG AA级以上）
- 避免纯黑纯白，使用灰度梯度提升体验
- 考虑色盲用户，不要仅用颜色传达信息

---

#### 3.2 页面列表 (Pages)
```
"pages": ["首页", "游戏大厅", "游戏界面"]
```

**页面设计原则**：
- 应包含3-8个核心页面
- 页面名称应清晰易懂
- 应按用户流程排序

**典型游戏产品页面**：
1. **首页/登录页**：品牌展示、用户进入
2. **大厅/主界面**：游戏发现、房间列表
3. **游戏界面**：核心游戏玩法
4. **个人中心**：用户信息、设置
5. **商店/社交**：虚拟物品、好友互动
6. **设置页**：音量、语言、隐私等

---

#### 3.3 组件设计 (Components)
```
[{
    "name": "组件名称",
    "type": "button|input|card|etc",
    "description": "组件描述"
}]
```

**组件类型参考**：
- **button**：可交互的按钮
- **input**：用户输入框（文本、数字等）
- **card**：内容容器（房间卡片、房产卡片）
- **toggle**：开关（设置选项）
- **slider**：滑块（数值调整）
- **list**：列表（房间列表、好友列表）
- **avatar**：头像（玩家标识）
- **icon**：图标（状态、功能标识）
- **modal**：对话框（确认、选择）
- **navigation**：导航（页面切换）
- **bar**：状态栏（资金、进度）
- **grid**：网格（棋盘、商品展示）
- **interactive**：交互组件（骰子、轮转）

**组件命名建议**：
✅ 好：「游戏开始按钮」、「房间卡片」、「玩家资金状态栏」
❌ 差：「按钮1」、「卡」、「状态」

---

#### 3.4 交互设计 (Interactions)
```
"interactions": ["点击按钮开始游戏", "拖拽棋子移动"]
```

**交互类型**：
- **点击/轻按**：按钮、菜单项
- **拖拽**：棋子移动、卡片排序
- **滑动**：列表滚动、翻页
- **长按**：上下文菜单、详情查看
- **双击**：快速操作
- **悬停**：桌面端提示
- **语音/手势**：特殊交互

**交互描述规范**：
✅ 好：「点击房产卡片弹出购买对话框」、「拖拽棋子到目标位置」
❌ 差：「用户交互」、「各种操作」

## 常见问题与解决方案

### 问题1：页面设计过于简单
**症状**：只有3-4个页面，缺少重要功能页面

**解决方案**：
1. 在提示词中明确"至少包含5-8个核心页面"
2. 列举必需的页面类型
3. 提供参考

```python
updated_prompt = f"""...
## 页面列表要求
请包含以下类型的页面：
- 首页/登录页（用户入口）
- 主界面/大厅（游戏发现）
- 游戏界面（核心玩法）
- 个人中心（用户管理）
- 社交/排行榜（互动）
- 商店（变现）
- 设置（配置）

总计至少7-10个页面
...
"""
```

---

### 问题2：颜色方案不够和谐
**症状**：颜色搭配不协调，缺少美感

**解决方案**：
1. 在提示词中指定颜色搭配原则
2. 提供参考调色板
3. 指定设计风格

```python
updated_prompt = f"""...
## 颜色方案要求
请设计符合以下原则的颜色方案：
- 采用互补色或相近色方案
- 主色应具有品牌识别度
- 确保亮度和饱和度的平衡
- 参考设计风格：现代简约|运动|高端|温暖

提供参考调色板：
- 运动风格：#FF6B6B (红) + #4ECDC4 (青) + #FFE66D (黄)
- 高端风格：#1A1A1A (深灰) + #D4AF37 (金) + #FFFFFF (白)
...
"""
```

---

### 问题3：组件设计不够完整
**症状**：缺少某些关键组件

**解决方案**：
1. 明确要求组件数量
2. 按功能分类列举必需组件
3. 提供检查清单

```python
updated_prompt = f"""...
## 组件设计检查清单
请确保包含：
- 操作类（按钮、输入框）：至少3个
- 展示类（卡片、列表、网格）：至少3个
- 反馈类（模态框、提示、进度条）：至少2个
- 导航类（标签页、菜单、导航栏）：至少2个
- 特殊类（骰子、棋盘、头像）：根据功能需要

总计至少10-15个组件
...
"""
```

### v1.0 - 初始版本

## 最佳实践

### 1. 清晰的设计目标
✅ 好：「为18-35岁的移动游戏玩家设计、年轻活力风格」
❌ 差：「设计一个好看的UI」

### 2. 具体的设计约束
✅ 好：「颜色方案使用互补色、确保AA级对比度」
❌ 差：「选好看的颜色」

### 3. 完整的组件清单
✅ 好：提供必需的组件类型和功能
❌ 差：「设计一些组件」

### 4. 用户为中心
✅ 好：强调目标用户和使用场景
❌ 差：仅关注美观性

### 5. 设计系统思维
✅ 好：考虑可复用性、一致性、可扩展性
❌ 差：逐个设计页面
