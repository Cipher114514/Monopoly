# Requirements Agent 提示词文档

## 概述
本文档记录了Requirements Agent（需求分析Agent）的提示词设计和演进历史。Requirements Agent负责分析用户需求，生成结构化的产品需求文档（PRD）。

---

## 核心提示词

### 主提示词模板

```python
requirements_prompt = f"""你是资深产品经理。

## 上下文记忆
{context}

## 当前需求
{user_request}

## 任务
生成产品需求文档（PRD），包含以下信息：
1. product_name: 产品名称
2. features: 功能列表（数组，至少包含核心功能）
3. target_users: 目标用户群体
4. success_criteria: 验收标准（数组）

## 重要
请直接输出纯 JSON 格式，不要使用 markdown 代码块包裹，不要添加任何其他文字。

示例格式：
{{"product_name": "...", "features": [...], "target_users": "...", "success_criteria": [...]}}
"""
```

---

## 提示词组件详解

### 1. 角色设定 (Role Setting)
```
你是资深产品经理。
```

**目的**：
- 为LLM设定专业的角色上下文
- 确保输出具有产品经理的思维方式
- 提升输出质量和专业性

**优化建议**：
- 可根据产品类型调整：游戏产品经理、SaaS产品经理等
- 强调特定领域经验可提升相关输出

---

### 2. 上下文记忆 (Context Memory)
```
## 上下文记忆
{context}

## 当前需求
{user_request}
```

**目的**：
- 提供历史信息和对话上下文
- 确保PRD与已知信息的一致性
- 支持迭代改进

**数据来源**：
- 来自`memory.summarize_context()`
- 包含长期记忆（产品信息）和短期记忆（最近对话）

**示例内容**：
```
## 已知信息：
- product_name: 在线大富翁对战平台
- features: [多人实时对战, 房产交易系统, ...]
- ui_pages: [首页, 游戏大厅, ...]

## 最近对话：
User: 创建一个在线大富翁游戏
...
```

---

### 3. 任务定义 (Task Definition)
```
## 任务
生成产品需求文档（PRD），包含以下信息：
1. product_name: 产品名称
2. features: 功能列表（数组，至少包含核心功能）
3. target_users: 目标用户群体
4. success_criteria: 验收标准（数组）
```

**目的**：
- 明确要求的输出字段
- 指定数据格式（数组、字符串等）
- 设定最低质量标准

**关键字段解释**：
- `product_name`: 简洁、记忆性强的产品名称
- `features`: 至少3-5个核心功能，应包含独特价值主张
- `target_users`: 明确的用户画像（年龄、职业、需求等）
- `success_criteria`: 可衡量的成功指标（用户数、留存率、满意度等）

---

### 4. 输出格式 (Output Format)
```
## 重要
请直接输出纯 JSON 格式，不要使用 markdown 代码块包裹，不要添加任何其他文字。

示例格式：
{{"product_name": "...", "features": [...], "target_users": "...", "success_criteria": [...]}}
```

**目的**：
- 强制LLM输出结构化数据
- 便于后续JSON解析
- 减少解析错误

**重要性**：
- 这部分对LLM的约束很关键
- 防止LLM输出额外解释或markdown格式
- 直接输出JSON便于程序处理

## 高级提示词技巧

### 1. Chain-of-Thought（思维链）增强版
```python
advanced_prompt = f"""你是资深产品经理。

## 用户需求
{user_request}

## 分析步骤
1. 理解用户需求的核心价值主张
2. 识别目标用户群体特征
3. 列举核心功能和差异化功能
4. 定义可衡量的成功标准

## 任务
基于以上分析，生成PRD JSON格式。
"""
```

**优势**：
- LLM先进行分析再输出
- 提升输出的逻辑性和完整性
- 适合复杂需求

---

### 2. 示例驱动提示词 (Few-Shot)
```python
few_shot_prompt = f"""你是资深产品经理。

## 已有示例
### 示例1：在线大富翁游戏
- 产品名称：在线大富翁对战平台
- 核心功能：多人对战、房产交易、实时同步
- 目标用户：18-45岁社交游戏爱好者
- 成功标准：10万DAU、30分钟平均游戏时长

### 示例2：项目管理工具
- 产品名称：团队协作任务系统
- 核心功能：任务管理、团队协作、进度跟踪
- 目标用户：中小企业项目经理
- 成功标准：5000B2B用户、90%留存率

## 新需求
{user_request}

请按照示例的格式和质量，生成新的PRD JSON。
"""
```

**优势**：
- 通过示例指导LLM的输出风格
- 提升输出质量和一致性
- 特别适合要求严格格式的场景

## 常见问题与解决方案

### 问题1：JSON解析失败
**症状**：LLM输出包含markdown代码块或额外文本

**解决方案**：
1. 强化提示词中的"纯JSON"要求
2. 在代码中添加清理逻辑
3. 使用正则表达式提取JSON

```python
# 代码中的处理示例
if response_text.startswith("```"):
    lines = response_text.split('\n')
    if lines[0].startswith("```"):
        lines = lines[1:]
    if lines[-1].startswith("```"):
        lines = lines[:-1]
    response_text = '\n'.join(lines).strip()
```

---

### 问题2：功能列表过于简陋
**症状**：features只有1-2项，缺少深度

**解决方案**：
1. 在提示词中明确"至少包含5-10个功能"
2. 区分核心功能、辅助功能、高级功能
3. 添加功能分类要求

```python
updated_prompt = f"""...
## 功能需求
请包含以下类型的功能：
- 核心功能（3-5个）：产品的核心价值主张
- 支持功能（2-3个）：辅助用户使用
- 高级功能（1-2个）：增强竞争力
- 社交功能（1-2个）：用户互动
...
"""
```

---

### 问题3：验收标准不可衡量
**症状**：success_criteria包含"易用性好"等模糊指标

**解决方案**：
1. 明确要求具体数字指标
2. 提供示例指标
3. 指定指标的衡量方式

```python
updated_prompt = f"""...
## 验收标准
请包含可衡量的KPI（Key Performance Indicator），如：
- 用户数量：DAU、MAU
- 留存指标：次日留存率、7日留存率
- 体验指标：平均游戏时长、满意度评分
- 商业指标：转化率、ARPPU

示例：
- 同时在线用户数达到10万
- 平均游戏时长超过30分钟
- 用户满意度评分不低于4.5分
...
"""
```

## 最佳实践

### 1. 清晰的角色设定
✅ 好：「你是资深产品经理，有15年的游戏行业经验」
❌ 差：「生成一个PRD」

### 2. 具体的示例
✅ 好：提供2-3个参考PRD示例
❌ 差：只说"生成PRD"

### 3. 明确的约束条件
✅ 好：「功能列表至少10项，验收标准必须包含数字指标」
❌ 差：「生成合理的验收标准」

### 4. 结构化的输出要求
✅ 好：指定JSON字段和数据类型
❌ 差：「输出相关信息」

### 5. 错误处理指导
✅ 好：「如果无法理解需求，请请求澄清」
❌ 差：没有说明处理异常情况的方式
