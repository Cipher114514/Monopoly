### 2.2 LLM (大语言模型) - Agent的大脑

#### 什么是LLM？
```
LLM = Large Language Model（大语言模型）

简单理解：
- 就是一个超级聪明的AI
- 你给它文字，它返回文字
- 它能理解、推理、生成内容

常见例子：
- ChatGPT (OpenAI)
- Claude (Anthropic)
- 文心一言 (百度)
```

#### Agent系统如何使用LLM？

```
工作流程：
1. 你的代码 → 发送请求给LLM API
2. LLM → 处理并返回结果
3. 你的代码 → 接收结果继续处理

类比：
- LLM = 一个超级聪明的顾问
- API = 你和顾问通话的电话
- Prompt = 你给顾问的问题
```

#### API调用示例

```python
# 示例：调用OpenAI API
import openai

# 设置API密钥
openai.api_key = "你的密钥"

# 调用
response = openai.ChatCompletion.create(
    model="gpt-4",
    messages=[
        {"role": "user", "content": "你好"}
    ]
)

# 获取回答
answer = response.choices[0].message.content
print(answer)  # 输出: 你好！有什么可以帮你的吗？
```

### 2.4 向量数据库 - Memory的核心

#### 什么是向量数据库？
```
普通数据库: 精确匹配
- 查询: name = "张三" ✓
- 查询: name like "张三%" ✓

向量数据库: 语义搜索
- 查询: "类似的用户需求"
- 返回: "添加购物车功能" (语义相似)

原理：
1. 文本 → 转换成数字向量（类似坐标）
2. 相似的内容，向量距离近
3. 可以找到语义相似的内容
```

#### 为什么需要向量数据库？
```
场景：Agent需要记住历史经验

不用向量DB：
- 只能精确匹配
- 无法找到相似的历史案例
- Memory能力弱

用向量DB：
- 可以搜索"类似的问题"
- 找到历史解决方案
- 智能检索
```

#### 主流向量数据库对比

| 数据库 | 难度 | 优点 | 缺点 | 推荐度 |
|-------|------|------|------|--------|
| **Chroma** | ⭐ | 最简单、嵌入式 | 功能较简单 | ⭐⭐⭐⭐⭐ |
| **FAISS** | ⭐⭐ | Meta出品、最快 | 需要自己管理 | ⭐⭐⭐ |
| **Pinecone** | ⭐ | 云服务、简单 | 需要付费、国外 | ⭐⭐⭐ |
| **Milvus** | ⭐⭐⭐ | 功能强大 | 复杂、重 | ⭐⭐ |

#### 推荐方案

**入门阶段：Chroma（最简单）**
```python

# 使用

# 创建数据库（自动创建本地文件）

# 添加数据

    documents=["需求1", "需求2"],
    ids=["req1", "req2"]

# 搜索

    query_texts=["类似的需求"],
    n_results=2
```

### 3.2 LangGraph - 最新最灵活

#### 代码示例（稍微复杂但强大）
```python
from langgraph.graph import StateGraph
from langchain_openai import ChatOpenAI

# 1. 定义状态（Agent间的共享数据）
class AgentState(TypedDict):
    messages: list
    current_agent: str

# 2. 定义每个Agent
def pm_agent(state: AgentState):
    # PM的逻辑
    response = llm.invoke("你是PM...")
    return {"messages": [response]}

def req_agent(state: AgentState):
    # 需求分析Agent的逻辑
    response = llm.invoke("你是需求分析师...")
    return {"messages": [response]}

# 3. 构建流程图
workflow = StateGraph(AgentState)

# 添加节点（Agent）
workflow.add_node("PM", pm_agent)
workflow.add_node("Requirements", req_agent)

# 添加边（流程）
workflow.add_edge("PM", "Requirements")

# 设置入口和出口
workflow.set_entry_point("PM")
workflow.set_finish_point("Requirements")

# 4. 编译成应用
app = workflow.compile()

# 5. 运行
result = app.invoke({"messages": []})
```

#### 优点
```
✓ 完全自定义流程
✓ 强大的状态管理
✓ 可视化流程图
✓ 基于LangChain，生态成熟
```

#### 缺点
```
✗ 学习曲线稍陡
✗ 需要理解LangChain
✗ 代码相对多一些
```

#### 适合场景
```
✓ 复杂的多Agent系统
✓ 需要灵活控制流程
✓ 生产环境部署
```
