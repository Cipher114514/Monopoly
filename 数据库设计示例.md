# 在线大富翁游戏 数据库设计示例

## 数据库选型
- 主数据库: PostgreSQL 15+
- 缓存数据库: Redis 7+
- 实时数据: Redis Pub/Sub

---

## 核心数据表设计

### 1. 用户表 (users)
存储系统用户基础信息

```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    avatar_url VARCHAR(500),
    total_games INTEGER DEFAULT 0,
    wins INTEGER DEFAULT 0,
    level INTEGER DEFAULT 1,
    experience INTEGER DEFAULT 0,
    is_online BOOLEAN DEFAULT FALSE,
    last_login_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_users_username ON users(username);
CREATE INDEX idx_users_email ON users(email);
```

---

### 2. 游戏房间表 (game_rooms)
存储游戏房间信息

```sql
CREATE TABLE game_rooms (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(100) NOT NULL,
    host_id UUID NOT NULL REFERENCES users(id),
    max_players INTEGER DEFAULT 4 CHECK (max_players >= 2 AND max_players <= 8),
    current_players INTEGER DEFAULT 0,
    game_status VARCHAR(20) DEFAULT 'waiting', -- waiting, playing, finished
    map_id INTEGER DEFAULT 1,
    starting_balance INTEGER DEFAULT 1500,
    turn_time_limit INTEGER DEFAULT 60,
    is_public BOOLEAN DEFAULT TRUE,
    password_hash VARCHAR(255),
    started_at TIMESTAMP,
    finished_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_game_rooms_status ON game_rooms(game_status);
CREATE INDEX idx_game_rooms_host ON game_rooms(host_id);
```

---

### 3. 游戏玩家表 (room_players)
存储房间内玩家关联关系

```sql
CREATE TABLE room_players (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    room_id UUID NOT NULL REFERENCES game_rooms(id),
    user_id UUID NOT NULL REFERENCES users(id),
    player_position INTEGER DEFAULT 0,
    balance INTEGER DEFAULT 1500,
    color VARCHAR(20) NOT NULL,
    is_host BOOLEAN DEFAULT FALSE,
    turn_order INTEGER NOT NULL,
    is_bankrupt BOOLEAN DEFAULT FALSE,
    joined_at TIMESTAMP DEFAULT NOW(),
    
    UNIQUE(room_id, user_id),
    UNIQUE(room_id, turn_order)
);

CREATE INDEX idx_room_players_room ON room_players(room_id);
CREATE INDEX idx_room_players_user ON room_players(user_id);
```

---

### 4. 地产属性表 (properties)
存储地图上的所有地产

```sql
CREATE TABLE properties (
    id INTEGER PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    type VARCHAR(50) NOT NULL, -- property, station, utility, chance, community, tax, go, jail
    position INTEGER UNIQUE NOT NULL,
    price INTEGER,
    rent_base INTEGER,
    rent_house1 INTEGER,
    rent_house2 INTEGER,
    rent_house3 INTEGER,
    rent_house4 INTEGER,
    rent_hotel INTEGER,
    house_cost INTEGER,
    hotel_cost INTEGER,
    mortgage_value INTEGER,
    color_group VARCHAR(20),
    board_side INTEGER
);
```

---

### 5. 玩家地产所有权表 (player_properties)
存储玩家拥有的地产

```sql
CREATE TABLE player_properties (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    room_id UUID NOT NULL REFERENCES game_rooms(id),
    property_id INTEGER NOT NULL REFERENCES properties(id),
    owner_id UUID REFERENCES room_players(id),
    houses INTEGER DEFAULT 0 CHECK (houses >= 0 AND houses <= 5),
    is_mortgaged BOOLEAN DEFAULT FALSE,
    updated_at TIMESTAMP DEFAULT NOW(),
    
    UNIQUE(room_id, property_id)
);

CREATE INDEX idx_player_properties_room ON player_properties(room_id);
CREATE INDEX idx_player_properties_owner ON player_properties(owner_id);
```

---

### 6. 游戏交易记录表 (game_transactions)
存储游戏内所有资金交易

```sql
CREATE TABLE game_transactions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    room_id UUID NOT NULL REFERENCES game_rooms(id),
    from_player UUID REFERENCES room_players(id),
    to_player UUID REFERENCES room_players(id),
    amount INTEGER NOT NULL,
    transaction_type VARCHAR(50) NOT NULL,
    description TEXT,
    turn_number INTEGER,
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_transactions_room ON game_transactions(room_id);
CREATE INDEX idx_transactions_from ON game_transactions(from_player);
```

## 数据库设计要点

1. **范式设计**: 第三范式，减少数据冗余
2. **外键约束**: 保证数据完整性
3. **索引优化**: 所有查询字段都建立适当索引
4. **时间字段**: 所有表包含创建/更新时间戳
5. **软删除**: 使用状态字段代替物理删除
6. **约束检查**: 数据范围校验在数据库层面实现

## 缓存设计

| 缓存Key | 类型 | 过期时间 | 说明 |
|---------|------|----------|------|
| `room:{id}` | Hash | 24小时 | 游戏房间实时状态 |
| `player:{room}:{user}` | Hash | 24小时 | 玩家实时状态 |
| `online_users` | Set | 实时 | 在线用户集合 |
| `game_turn:{room}` | String | 60秒 | 当前回合信息 |
