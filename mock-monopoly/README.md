# Testing Agent - 生产环境代码测试工具

## 📋 概述

**Testing Agent** 是一个基于 LLM 的自动化代码测试工具，用于真实项目的代码分析和问题发现。

### 核心功能

1. ✅ **智能代码分析** - 读取并分析项目代码
2. ✅ **自主发现 Bug** - 无需预设，自动发现所有潜在问题
3. ✅ **多维度测试** - 功能、安全、性能、架构、代码质量
4. ✅ **自动生成报告** - 生成完整的测试报告和修复建议
5. ✅ **实时进度显示** - 可视化测试进度和结果

## ✨ 核心特性

### 🔍 API 预检查
- 自动检查 API Key 格式
- 验证网络连接状态
- 测试 API 可用性
- 提前发现问题，避免长时间等待

### 📊 实时进度显示
- 逐文件显示读取进度
- 显示每个文件的行数、字符数
- 汇总统计信息（总行数、总字符数）
- API 调用实时耗时显示
- Token 使用情况统计

### 🤖 智能 Bug 组织
- **自动生成 Bug ID** - 按发现顺序编号（BUG-001, BUG-002...）
- **严重程度分级** - P0/P1/P2/P3 自动分级
- **问题类型分类** - 功能/安全/性能/架构/代码质量
- **修复建议** - 针对每个问题提供具体修复方案

### 🔄 智能重试机制
- 最多重试 3 次
- 自动识别超时、网络错误
- 区分可重试错误和不可重试错误（如认证失败）

## 🚀 如何使用

### 快速开始

```bash
# 运行 Testing Agent
python testing_agent.py
```

### 运行流程

1. **预检查 API 可用性**
   - 检查 API Key 格式
   - 验证网络连接
   - 测试 API 调用

2. **读取代码文件**
   - 读取所有后端代码
   - 实时显示读取进度
   - 统计代码行数和字符数

3. **LLM 代码分析**
   - 发送代码到智谱 AI
   - 多维度分析（功能、安全、性能、架构、代码质量）
   - 自动发现所有问题

4. **生成测试报告**
   - 自动生成 Bug ID
   - 按严重程度分级（P0/P1/P2/P3）
   - 按类型分类
   - 提供修复建议
   - 保存完整报告到文件


### 2. 保存测试报告

```bash
python test_testing_agent_real_llm.py > test_report_$(date +%Y%m%d_%H%M%S).txt 2>&1
```

## ❓ 常见问题

### Q1: 运行时报错 "ModuleNotFoundError"

**原因**：缺少 requests 库

**解决**：
```bash
pip install requests
```

### Q2: API 调用失败

**原因**：网络问题或 API Key 失效

**解决**：
1. 检查网络连接
2. 确认 API Key 有效
3. 查看 API 剩余额度

### Q3: 找不到预设的 Bug

**原因**：
1. 代码文件被修改
2. LLM 解析不准确

**解决**：
1. 检查代码文件是否完整
2. 重新运行测试
3. 查看 LLM 返回的完整报告
