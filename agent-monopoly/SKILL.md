# 在线大富翁游戏 - Agent技能说明

## 概述

本项目使用多Agent协作团队开发，每个Agent负责软件开发生命周期的不同阶段。

## Agent技能列表

### 1. PM Agent（产品经理）
**技能**: 产品规划、需求分析、优先级排序

**输入**: 用户需求、《需求规格说明书》
**输出**: PRD（产品需求文档）

**核心能力**:
- 分析17个功能需求
- 定义核心游戏规则
- 制定2周开发计划
- 用户画像分析

**验证要点**:
- 是否覆盖17个功能需求
- 核心规则是否完整
- 时间规划是否合理

---

### 2. Requirements Agent（需求分析师）
**技能**: 需求分解、任务规划、RICE评分

**输入**: PRD
**输出**: 任务列表（Sprint规划）

**核心能力**:
- 将17个功能需求转化为具体任务
- 设置优先级（P0/P1）
- 估算工时
- 定义依赖关系

**验证要点**:
- 是否覆盖所有功能需求
- 任务粒度是否合适
- 依赖关系是否正确

---

### 3. Architecture Agent（架构师）
**技能**: 系统架构、技术选型、数据库设计

**输入**: PRD、任务列表
**输出**: 架构设计文档

**核心能力**:
- 设计服务端权威架构
- 选择技术栈（Node.js + SQLite + Socket.io）
- 设计7张核心表
- 定义API接口

**验证要点**:
- 是否支持17个功能需求
- 架构是否防作弊
- 数据库是否支持回放

---

### 4. Design Agent（系统设计师）
**技能**: 详细设计、算法设计、ADR记录

**输入**: 架构设计
**输出**: 系统设计文档

**核心能力**:
- 记录架构决策（ADR）
- 设计核心算法：
  - 破产算法（递归变卖）
  - 回合流转算法
  - 过路费计算算法
  - 卡牌队列算法
- 设计数据流

**验证要点**:
- 算法是否正确
- 是否贴合需求
- 是否可实现

---

### 5. Coding Agent（高级开发者）
**技能**: 代码实现、模块设计、最佳实践

**输入**: 系统设计
**输出**: 代码实现

**核心能力**:
- 实现17个功能需求
- 实现9个核心游戏规则：
  1. 路过起点发钱，停在起点不发钱
  2. 只能在停着的普通地产上盖房
  3. 特殊地块不能盖房
  4. 升级顺序：空地→房子→2房子→旅馆
  5. 有建筑的土地不能抵押
  6. 抵押期间不收过路费
  7. 破产前先变卖资产
  8. 卡牌队列循环使用
  9. 进牢/坐牢规则

**验证要点**:
- 17个功能需求是否全部实现
- 9个核心规则是否正确
- 代码质量是否达标

---

### 6. Code Review Agent（代码审查员）
**技能**: 代码审查、最佳实践、问题识别

**输入**: 代码实现
**输出**: 代码审查报告

**核心能力**:
- 审查代码质量
- 识别逻辑错误
- 检查需求贴合度
- 提供改进建议

**审查重点**:
- 17个功能需求是否实现
- 9个核心规则是否正确
- 是否有阻塞问题

**验证要点**:
- 是否发现所有逻辑错误
- 建议是否可操作
- 评分是否合理

---

### 7. Testing Agent（测试工程师）
**技能**: 测试设计、功能测试、Bug识别

**输入**: 代码实现
**输出**: 测试报告

**核心能力**:
- 设计测试用例
- 执行功能测试
- 验证9个核心规则
- 记录Bug

**测试重点**:
- 17个功能需求的测试
- 9个核心规则的验证

- 测试覆盖率是否达标
- Bug描述是否清晰

---

### 8. QA Agent（质量保证工程师）
**技能**: 质量评估、验收测试、决策建议

**输入**: 测试报告、代码实现
**输出**: QA评估报告

**核心能力**:
- 评估整体质量
- 验证需求贴合度
- 判断是否可以答辩

**评估维度**:
- 功能完成度（17个FR）

- 答辩准备度

**验证要点**:
- 评估是否客观
- 建议是否实用
- 是否符合课程要求

---

### 9. Documentation Agent（技术文档工程师）
**技能**: 文档编写、技术沟通、知识管理

**输入**: 所有阶段的输出
**输出**: 技术文档

**核心能力**:
- 编写README
- 编写架构文档
- 编写API文档
- 编写需求贴合说明

**文档清单**:
1. README.md

3. API.md

**验证要点**:
- 文档是否完整
- 是否清晰易懂
- 是否覆盖所有内容

## 核心游戏规则验证清单

所有Agent都必须验证以下9个核心游戏规则：

### 1. 起点规则
- [ ] 路过起点发钱
- [ ] 停在起点不发钱

### 2. 盖房规则
- [ ] 只能在停着的普通地产上盖房
- [ ] 路过不能盖房

### 3. 特殊地块规则
- [ ] 特殊地块（电站、车站、水厂）不能盖房

### 4. 建设规则
- [ ] 升级顺序：空地 → 房子 → 2房子 → 旅馆

### 5. 抵押规则
- [ ] 有建筑的土地不能抵押
- [ ] 抵押期间不收过路费

### 6. 破产规则
- [ ] 先变卖资产（房子半价、土地半价抵押）
- [ ] 变卖所有资产后仍不够，才破产
- [ ] 破产后所有地产回归空白

### 7. 卡牌规则
- [ ] 按队列顺序拿卡（队首）
- [ ] 执行后移至队尾（循环）

### 8. 坐牢规则
- [ ] 进牢格：直接移动到坐牢格，暂停1回合
- [ ] 坐牢格：路过无惩罚

### 9. 免费停车规则
- [ ] 免费停车场：暂停1回合

---

## 技能验证流程

### 1. 自我验证
每个Agent在输出前进行自我验证：
- 是否完成了自己的职责
- 是否贴合需求规格说明书
- 是否验证了核心规则

### 2. 交叉验证
下一个Agent验证上一个Agent的输出：
- PM Agent → Requirements Agent
- Requirements Agent → Architecture Agent
- ...
- Documentation Agent（最后验证所有）

### 3. 最终验证
QA Agent进行最终验证：
- 17个功能需求是否完成
- 9个核心规则是否正确
- 是否可以参加答辩

---

## 输出质量标准

### 优秀 (A)
- 17个功能需求全部实现
- 9个核心规则全部正确
- 代码质量高，文档完整

### 良好 (B+)
- 17个功能需求基本实现
- 9个核心规则大部分正确
- 有小问题但不影响演示

### 及格 (B)
- 17个功能需求部分实现
- 9个核心规则部分正确
- 有较明显问题但可演示

### 不及格 (C)
- 17个功能需求大量缺失
- 9个核心规则多数错误
- 无法进行演示
