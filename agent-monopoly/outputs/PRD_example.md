# PRD: 在线大富翁 Web 版（MVP）

## 1. Problem Statement（问题陈述）
学生群体（18-22岁）希望在碎片时间与好友线上玩经典大富翁，但传统桌游受制于地点与时间，学习成本高且难以实时同步玩家状态。

## 2. Goals & Success Metrics（目标与成功指标）
| Goal | Metric | Current Baseline | Target |
|------|--------|------------------|--------|
| 提供稳定的多人实时对战体验 | 平均延迟（ms） | 未测量 | <200ms |
| 提升次日留存 | 次日留存率 | 未测量 | >=40% |
| 提高平均单局时长 | 平均游戏时长 | 未测量 | >=8 分钟 |

## 3. User Personas & Stories
**Primary Persona**: 大学用户（18-22岁），偏好社交、时间碎片化、追求快节奏体验

核心用户故事：
- 作为一名玩家，我希望能快速创建/加入房间，以便立即开始游戏。
- 作为房主，我希望能配置房间（人数、地图、规则），以便与朋友自定义玩法。

验收标准示例：
- Given 已登录用户，when 点击“创建房间”，then 房间被创建并进入等待页面。

## 4. Solution Overview
实现一套基于 WebSocket 的前后端分离架构，后端负责权威游戏逻辑与状态同步，前端以轻量交互界面呈现地图与玩家操作。初期只实现 Web 平台，支持 2-6 人实时对战、基础聊天与断线重连。

关键设计决策：
- 使用 WebSocket（Socket.io）保证实时性，后端采用 Node.js 服务端权威逻辑以防作弊。

## 5. Technical Considerations
Dependencies:
- 用户认证服务、实时通信（WebSocket）、数据库持久化、静态资源托管

Known Risks:
- 状态同步错位导致回合不同步（缓解：权威服务器+操作确认机制）
- 高并发下延迟升高（缓解：性能测试+限流+CDN）
